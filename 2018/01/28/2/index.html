0X11 编码注入

编码处理的函数存在问题，输入与转码函数不兼容的特殊字符，可以导致输出的字符变成有害的数据。
常见的是宽字节注入和urldecode/ rawurldecode函数导致的。

宽字节注入

条件：php连接MySQL时，设置 set character_set_client=gbk。
参数带入 %df%27 ，可以把过滤的 \ (%5c) 组合成 %df %5c，即单引号没有被转义。

解决办法：

白盒关键字：SET NAMES，set character_set_client=gbk，mysql_set_charset(‘gbk’)

二次urldecode注入

?id=1%2527
?id=1%27
?id=1’

关键字：urldecode， rawurldecode
